Proyecto Final - Sistemas Operativos

Implementación de un servidor HTTP utilizando GitHub Pages para demostración de conceptos de sistemas operativos

Acerca del Proyecto

Este proyecto final de sistemas operativos demuestra cómo configurar y utilizar un servidor HTTP básico, aprovechando GitHub Pages como entorno de hosting. La implementación ilustra conceptos clave de administración de sistemas, redes y servidores.

Desarrollo Web

Creación de una página web estática utilizando HTML5, CSS3 y JavaScript para la interfaz de usuario.

Servidor HTTP

Configuración de un entorno de servidor web utilizando GitHub Pages como plataforma de hosting.

Sistemas Operativos

Aplicación de conceptos de administración de sistemas, procesos, memoria y sistemas de archivos.

Configurar GitHub Pages como Servidor HTTP

GitHub Pages permite alojar sitios web estáticos directamente desde un repositorio de GitHub. A continuación, se detallan los pasos para configurarlo:

  1. Crear un repositorio en GitHub

    Crea un nuevo repositorio público en GitHub con el nombre "nombreusuario.github.io" (reemplaza "nombreusuario" con tu nombre de usuario de GitHub).

  2. Subir los archivos del proyecto

    Sube todos los archivos HTML, CSS, JavaScript y recursos a la raíz del repositorio. El archivo principal debe llamarse "index.html".

  3. Configurar GitHub Pages

    Ve a la configuración del repositorio, desplázate hasta la sección "GitHub Pages" y selecciona la rama "main" como fuente.

  4. Acceder al sitio web

    Tu sitio estará disponible en "https://nombreusuario.github.io" después de unos minutos. GitHub actuará como tu servidor HTTP.

Comandos Git para subir el proyecto

# Inicializar repositorio local
git init

# Agregar todos los archivos del proyecto
git add .

# Hacer commit de los cambios
git commit -m "Primera versión del proyecto final"

# Conectar con el repositorio remoto en GitHub
git remote add origin https://github.com/tuusuario/tuusuario.github.io.git

# Subir el código a GitHub
git push -u origin main

Conceptos de Sistemas Operativos Aplicados

Este proyecto implementa y demuestra varios conceptos clave de sistemas operativos:

Gestión de Procesos

El servidor web maneja múltiples solicitudes HTTP concurrentes, similar a cómo un SO gestiona múltiples procesos.

Administración de Memoria

GitHub Pages asigna recursos de memoria para servir contenido estático, ilustrando conceptos de gestión de memoria.

Sistema de Archivos

La estructura de directorios y archivos en el repositorio refleja cómo los SO organizan y acceden a los datos.

Seguridad y Permisos

GitHub implementa controles de acceso y permisos para los repositorios, similar a los mecanismos de seguridad de un SO.
